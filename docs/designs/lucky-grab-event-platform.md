## Vision

### 10x Check
TAPE의 모든 오프라인 이벤트를 하나의 플랫폼으로 운영하는 "이벤트 엔진":
- 관리자가 이벤트 생성 → 보상 풀 설정 → QR 자동 생성
- 손님이 QR 스캔 → 게임 플레이 → 서버가 당첨 판정 → 쿠폰 즉시 발급
- 쿠폰은 유효기간 스탑워치 + 사용완료 버튼으로 매장에서 직접 처리
- 실시간 대시보드: 참여율, 남은 보상 수량, 시간대별 트래픽
- 이벤트 만료 시 자동 비활성화, 결과 리포트 생성

### Platonic Ideal
손님이 매장 QR을 스캔하는 순간부터의 경험:
1. **0초**: QR 스캔 → 로딩 없이 즉시 게임 화면 (< 1초)
2. **5초**: 직관적으로 크레인 조작 시작 (설명서 불필요)
3. **15초**: 공을 잡는 순간 — 햅틱 진동 + "과연...?" 긴장감 연출
4. **20초**: 결과 공개 — "10% 할인 쿠폰 당첨!" + 쿠폰 화면
5. **25초**: SNS 공유 카드로 인스타/카톡에 자랑
6. **30초**: 쿠폰 카운트다운 시작 — 매장에서 바로 사용

느낌: "와, 이 가게 재밌다." → 브랜드 기억 → 재방문

## Tech Stack
- **Frontend**: 순수 HTML/CSS/JS (프레임워크 없음)
- **Backend**: Supabase (PostgreSQL + Auth + Edge Functions)
- **Deploy**: GitHub Pages (프론트엔드) + Supabase (백엔드)

## Architecture

```
  GitHub Pages                              Supabase
  ─────────────                            ──────────
  index.html (게임)  ──┐                  ┌── Edge Function: /api/grab
  admin.html (관리자) ──┼── Supabase SDK ──┤── Auth (관리자용)
  coupon.html (쿠폰) ──┘                  └── PostgreSQL (events, prizes,
                                                participants, coupons)
```

## DB Schema

```sql
-- events: 이벤트 관리
events (
  id uuid PK,
  name text,
  slug text UNIQUE,          -- QR URL에 사용
  starts_at timestamptz,
  ends_at timestamptz,
  max_plays_per_device int,  -- 기기당 참여 횟수 제한
  is_active bool,
  created_at timestamptz
)

-- prizes: 이벤트별 보상
prizes (
  id uuid PK,
  event_id uuid FK → events,
  title text,
  description text,
  weight int,                -- 당첨 확률 가중치
  total_quantity int,        -- null = 무제한
  remaining_quantity int,
  coupon_validity_minutes int
)

-- participants: 참여 기록 (어뷰징 방지)
participants (
  id uuid PK,
  event_id uuid FK → events,
  fingerprint text,
  ip_address inet,
  played_at timestamptz,
  prize_id uuid FK → prizes (nullable)
)

-- coupons: 발급된 쿠폰
coupons (
  id uuid PK,
  participant_id uuid FK → participants,
  prize_id uuid FK → prizes,
  code text UNIQUE,
  expires_at timestamptz,
  is_used bool DEFAULT false,
  used_at timestamptz,
  created_at timestamptz
)
```

## Critical Implementation Notes

### Race Condition 방지 (재고)
```sql
UPDATE prizes
SET remaining_quantity = remaining_quantity - 1
WHERE id = $1 AND remaining_quantity > 0
RETURNING *;
```
영향받은 행 = 0이면 해당 상품 소진 처리.

### 쿠폰 UX
1. **활성**: 쿠폰 정보 + 남은 시간 스탑워치 (카운트다운) + "사용완료" 버튼
2. **사용완료**: 회색/줄 그음 + "사용 완료됨" 텍스트
3. **만료**: "쿠폰이 만료되었어요" + 비활성화

### 보안 원칙
- anon key만 클라이언트에 노출 (RLS로 보호)
- service_role key는 Edge Function 환경변수에서만
- 관리자 DB 조작은 RLS로 auth.uid() 기반 제한
- 게임 참여자는 인증 불필요 (로그인 없이 즉시 플레이)
